# 500 BEST MOVIES

Рандомайзер топ-500 фильмов [Letterboxd](https://letterboxd.com/). Жмёшь **«ПОЙМАТЬ»** — станция
ловит из эфира случайный фильм-«канал», открываешь его на Letterboxd и собираешь коллекцию
просмотренного. Полностью статичное приложение: без своего сервера рендеринга и без оплаты хостинга.

🎬 **Живой сайт:** <https://films.masterofthestar.ru/>

Дизайн — вселенная 90-х: аркадный автомат, CRT-кинескоп, «космический телевизор». Каждый бросок —
это **открытие, а не «мимо»**: любой фильм = найден сигнал, топ-100 = ★, топ-10 = ◆ победа с залпом.

## Возможности

- 🎛 **Пульт слежения** — телеметрия, вращающийся каркасный глобус со спутником, аркадные гейджи прогресса.
- 📺 **Захват с драматургией** — растровые помехи, разгон до «резонанса», вспышка и пружинное раскрытие карточки-досье.
- 🎧 **Оператор-станции** — пиксельная героиня (спрайты в стиле PC-98), реагирует на события: приветствует при заходе, ищет, ликует по тиру, удивляется уже собранному, живёт в покое.
- 🗄 **Коллекция** — полноэкранная «видеостена»: 500 мониторов, пойманные светятся постером, непойманные — тёмный шум; магнитный курсор, режим «ВСЕ / МОИ».
- 🖼 **Постеры всех 500 фильмов** — самохостинг (см. ниже), работают в РФ без VPN.
- 🔄 **Синхронизация коллекции** между устройствами — анонимно, без аккаунтов (ссылка/код).
- 📱 **PWA** — ставится на телефон, работает офлайн.
- 🔀 **Диапазон поиска** — ловить из всех, только новых (без своих) или только из коллекции.
- 🌙 Уважает мобильные форматы, клавиатуру и фокус.

## Технологии

Один статичный `index.html` (стили, логика и данные внутри) — **ванильные HTML/CSS/JS, без сборки и фреймворков.**
Плюс два бесплатных «сервера по расписанию/запросу»:

- **GitHub Pages** — хостинг статики.
- **GitHub Actions** — раз в неделю обновляет базу и дозакачивает постеры.
- **Cloudflare Worker + KV** — крошечный анонимный «ящик» для синхронизации коллекции.

## Как устроены данные

База грузится по цепочке надёжности (что доступно первым — то и используется, постеры домёрживаются по `id`):

1. **Живой список** — свежий топ-500 у [bigbadraj/Letterboxd-List-JSONs](https://github.com/bigbadraj/Letterboxd-List-JSONs) (открытый CORS, тянется прямо из браузера).
2. **Локальные снапшоты** — `data/movies.enriched.json` (с постерами) → `data/movies.json`.
3. **Офлайн-фолбэк** — снимок всех 500 фильмов, вшитый в `index.html` (работает даже по двойному клику).

### Постеры

TMDB в России заблокирован по GeoDNS (включая `image.tmdb.org`), поэтому постеры **не хотлинкятся, а
скачиваются к себе**: скрипт `scripts/enrich-posters.mjs` запускается на серверах GitHub Actions (они не в РФ,
блокировки нет), кладёт картинки в `posters/<id>.jpg` и пишет пути в `data/movies.enriched.json`. Сайт отдаёт
их со своего адреса — видно всем, работает офлайн.

> Использует TMDB API, но не одобрен и не сертифицирован TMDB.

### Синхронизация

Опциональна и анонимна. Приложение генерит случайный «номер-ячейку», коллекция хранится в Cloudflare KV
под этим номером (ни имени, ни почты — это не персональные данные). Перенос между устройствами — по ссылке/QR,
дальше синк идёт сам. Код воркера — `cloudflare/sync-worker.js`, адрес прописывается в `config.js`.

## Структура

```
index.html                     всё приложение (UI, логика, офлайн-снапшот базы)
config.js                      адрес Cloudflare-воркера синхронизации
manifest.json  sw.js  icons/   PWA: манифест, service worker, иконки
data/movies.json               снапшот базы (обновляет бот)
data/movies.enriched.json      база с постерами (генерит бот)
posters/                       скачанные постеры (генерит бот)
sprites/                       спрайты оператора (op-*.jpg)
scripts/enrich-posters.mjs     обогащение постерами через TMDB
cloudflare/sync-worker.js      воркер синхронизации
.github/workflows/update-db.yml еженедельное автообновление
```

## Локальный запуск

Открыть `index.html` двойным кликом — работает офлайн (от вшитого снимка). Для полноценной разработки
(живые данные, постеры, синк, PWA) нужен http-сервер:

```bash
python -m http.server 8000
# затем открыть http://localhost:8000
```

## Деплой

1. Залить содержимое папки в репозиторий GitHub.
2. **Settings → Pages** → Deploy from a branch → `main` / `/root`.
3. (Опционально) свой домен: в **Settings → Pages → Custom domain** + CNAME-запись у провайдера DNS.

**Постеры / автообновление:** добавь секрет `TMDB_API_KEY` (Settings → Secrets → Actions), включи
запись для Actions (Settings → Actions → General → Read and write permissions) и запусти workflow
**update-db** вручную (вкладка Actions) — он скачает постеры и закоммитит. Дальше — раз в неделю сам.

**Синхронизация:** развернуть `cloudflare/sync-worker.js` как Cloudflare Worker с KV-биндингом `SYNC`
и прописать его адрес в `config.js`.
