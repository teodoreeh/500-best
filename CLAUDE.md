# 500 BEST — рандомайзер топ-500 фильмов Letterboxd

Контекст для Claude Code. Прочитай целиком перед изменениями. Автор — Фёдор («Хозяин Звезды»), арт-директор, не программист. Объясняй решения по-человечески, не ломай утверждённый дизайн.

## Что это

Веб-приложение: жмёшь «КРУТИТЬ» — выпадает случайный фильм из топ-500 Letterboxd. Можно открыть его на Letterboxd и добавить в личную коллекцию. Раньше это был Telegram-бот на FastAPI+SQLite (платный сервер). Цель переезда — **полностью статичное приложение без сервера и без оплаты**.

## Архитектура (принципы, не нарушать)

- **Ноль бэкенда.** Вся логика в браузере. Никаких FastAPI/SQLite/серверов.
- **Хранение коллекции** — `localStorage` (в `index.html` уже есть обёртка `store` с откатом в память). Никаких БД.
- **База фильмов** — статический JSON. Источник обновляется вручную/по расписанию, не в рантайме на сервере.
  - Первоисточник: `https://raw.githubusercontent.com/bigbadraj/Letterboxd-List-JSONs/main/film_titles_letterboxds-top-500-films.json` (автор репозитория обновляет список сам, отдаётся с открытым CORS — можно тянуть прямо из браузера).
  - В репозитории лежит снапшот `data/movies.json` как запасной вариант.
- **Постеры** — через TMDB, разовым build-скриптом (`scripts/enrich-posters.mjs`), результат запекается в JSON. Не дёргать TMDB в рантайме. Обязательна атрибуция TMDB в UI.
- **Хостинг** — Cloudflare Pages или GitHub Pages (бесплатно, статика). Деплой из гита.
- **Домен** — поддомен `films.masterofthestar.ru`. Сейчас A-запись ведёт на платный сервер Selectel (`111.88.119.182`); при переезде её надо сменить на CNAME к статик-хостингу, после чего сервер Selectel гасится.

## Дизайн-язык (утверждено, менять только по прямой просьбе автора)

Вселенная проекта — **90-е, аркадный автомат, CRT-кинескоп, космический телевизор**. Это НЕ личный бренд автора (сургучный #922D26) — его здесь не используем.

**Палитра** (CSS-переменные уже в `index.html`):
- фон `--bg:#05060E`, панели `--bg-2:#0A0D1C`, `--bg-3:#111633`, линии `--line:#1D2547`
- акценты: `--cyan:#19E3E3`, `--magenta:#FF2E97`, `--gold:#FFC24B`, `--violet:#9B6BFF`
- текст `--ink:#EAF2FF`, приглушённый `--dim:#6E77A0`
- кружки Letterboxd: `--lb-g:#00E054`, `--lb-o:#FF8000`, `--lb-b:#40BCF4`

**Шрифты:** JetBrains Mono (весь UI/данные), Press Start 2P (только маркиза «500 BEST»).

**Моушн и тактильность:** кейкап-кнопка с продавливанием; раскрытие фильма — пружина с перелётом (`cubic-bezier(.34,1.56,.64,1)`); скрембл-«поиск сигнала» перед раскрытием; удар-кадр (вспышка + микро-тряска); частицы. Уважать `prefers-reduced-motion`.

**Лестница награды (ключевая философия — не превращать в проигрыш):** каждый бросок — это открытие, а не «мимо».
- любой фильм (ранг 101–500) = «◦ НАЙДЕН СИГНАЛ», циан, лёгкий праздник;
- топ-100 (11–100) = «★ ТОП-100», маджента, ярче + тряска;
- топ-10 (1–10) = «◆ ТОП-10 · ПОБЕДА», золото, полный залп частиц + фанфара.

**Утверждённые решения (не откатывать без спроса):**
- Кость — плоский глиф d12, **без 3D** (фотореалистичная 3D-кость конфликтует с плоским CRT).
- Кнопка называется **«КРУТИТЬ»** (метафора барабана); глиф d12 крутится и делает пружинный доскок.
- Постеры — **чистые, единого размера, без CRT-фильтров** (фильтры «прожёвывали» постер, автору не зашло). Сейчас в ротации 3 демо-постера (ранги 3, 14, 165), вшиты base64. Заглушка для фильмов без постера — карточка того же размера с рангом.
- Коллекция — **полка VHS-кассет**: корешок в цвете тира, кремовая наклейка (ранг/название/год, до 2 строк), кнопка-пилюля с кружками Letterboxd, staggered-выезд при открытии, ховер «вытягивает» кассету.
- Прогресс — аркадные гейджи: ТОП-10 (10 сегментов), ТОП-100 и ВСЁ (светящиеся бары).
- «Фильм дня» — детерминированно по дате, показывается сам при загрузке (без отдельной кнопки).

**Чего НЕ делать:** не возвращать сургучный; не делать 3D-кость; не прогонять постеры через фильтры; не строить очки/бейджи/лидерборды (PBL) и стрик-механику вины — только честный прогресс; не делать так, чтобы неудачный ранг ощущался проигрышем.

## Структура файлов

- `index.html` — всё приложение (стили, логика, данные и 3 постера вшиты). Работает по двойному клику.
- `data/movies.json` — снапшот базы 500 фильмов (формат: `ListNumber`, `Title`, `Year`, `ID`; ссылка `letterboxd.com/film/{ID}/`).
- `scripts/enrich-posters.mjs` — стартовый скрипт обогащения постерами через TMDB (доделать).
- `.github/workflows/update-db.yml` — стаб автообновления базы с GitHub по расписанию.

## Дорожная карта (порядок)

1. **Разъединить данные.** Вынести базу из `index.html` в `data/movies.json`, грузить `fetch`-ем (для локальной разработки — простой статик-сервер). Оставить крошечный инлайн-фолбэк или снапшот.
2. **Живая база.** При загрузке пробовать свежий JSON с `raw.githubusercontent.com` (bigbadraj), при ошибке — локальный снапшот.
3. **Постеры TMDB.** Доделать `enrich-posters.mjs`: по `Title`+`Year` находить фильм в TMDB, брать `poster_path`, писать `data/movies.enriched.json` (или ссылки на CDN TMDB `image.tmdb.org`). Подключить в UI для всех 500, заглушка — если постера нет. Добавить атрибуцию TMDB.
4. **PWA + офлайн.** `manifest.json`, service worker, самохостинг шрифтов — чтобы ставилось на телефон и работало офлайн.
5. **Деплой.** Cloudflare Pages из репозитория, затем поддомен `films.masterofthestar.ru` (CNAME), погасить сервер Selectel.
6. **Автообновление.** GitHub Actions cron: раз в неделю тянуть свежий список bigbadraj и коммитить (+ по желанию до-обогащать постерами).

## Проверки

- Работает по двойному клику и после деплоя.
- Коллекция сохраняется между сессиями (`localStorage`).
- Мобилка (узкий экран) и клавиатура/фокус — ок.
- `prefers-reduced-motion` отключает тяжёлый моушн.
